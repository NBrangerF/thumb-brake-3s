# Hook 理论：Thumb Brake 3s 的拆解方式

[English](hook-theory.md) · [中文](hook-theory.zh-CN.md) · [Español](hook-theory.es.md)

> 你的商品只有 **3 秒**。
>
> 好的 Hook 不是请求用户关心，而是让“关心”变得立刻发生。

Thumb Brake 3s 建立在一个判断上：短视频 Hook 不是一句聪明的开场白，而是一个**被时间压缩过的注意力系统**。它必须让用户停滑、证明相关性，并在用户重新划走之前，把注意力自然承接到产品。

这份文档解释我们如何理解、拆解和生成 Hook。

---

## 1. 最短定义

Hook 是一个 **3 秒注意力契约**。

```text
0–1s   停住拇指
1–3s   证明“这和我有关”
3–7s   承接到产品
```

第一秒制造打断。  
接下来两秒证明值得继续看。  
之后几秒把注意力转化成对产品有意义的场景。

弱 Hook 说：

> “这里有一个产品。”

强 Hook 说：

> “这个瞬间已经发生在你的生活里，而这个产品正好属于这个瞬间。”

---

## 2. 美学前提

短视频注意力首先是身体性的。

用户一开始并不是在理性评估内容，而是在信息流里感受到一个很小的阻力：

- 一只手突然进入画面
- 一个物体出现反常行为
- 一句话点名了自己的具体处境
- 一个声音打断了滑动节奏
- 一个前后对比具体到无法忽略
- 一个社交场景像是正在真实发生

好的 Hook 像是让拇指经过一个小小的减速带。

项目名是字面意义上的：**Thumb Brake 3s** 试图制造那个让用户拇指慢下来的瞬间，让意义有时间抵达。

---

## 3. 技术前提

在这个项目里，Hook 不是松散文案，而是一个结构化创意资产。

一个 Hook 至少需要七个组件：

| 组件 | 作用 | 示例 |
|---|---|---|
| 停滑信号 | 打断信息流惯性 | 突然特写、异常动作、尖锐句子、静音切入 |
| 相关性证明 | 说明为什么用户要在意 | 可识别的日常、身份、痛点、愿望或处境 |
| 张力 | 给用户一个继续看的理由 | 某件事出错、未完成、反常或等待解释 |
| 场景证据 | 让 Hook 可被看见 | 道具、手势、环境、时间点、表情、屏幕状态 |
| 开放环 | 暂时延迟答案 | “等等，为什么会这样？” |
| 产品承接 | 把注意力接到产品 | 产品作为工具、证据、捷径或解决方式进入 |
| 边界约束 | 保持可信和安全 | 不虚假证明、不夸大医疗效果、不承诺平台定向 |

只有这些组件互相连接时，生成出来的 Hook 卡才真正有用。

---

## 4. 3 秒时间结构

核心结构不是“开头 → 正文 → CTA”，而是更压缩的注意力路径。

```text
┌───────────────┬──────────────────────────┬──────────────────────────────┐
│ 0–1s          │ 1–3s                    │ 3–7s                         │
│ 停滑信号       │ 相关性证明                │ 产品承接                       │
├───────────────┼──────────────────────────┼──────────────────────────────┤
│ 让用户停一下。  │ 让用户感觉：              │ 让产品显得自然、有用、           │
│               │ “这说的是我。”            │ 值得试一下。                    │
└───────────────┴──────────────────────────┴──────────────────────────────┘
```

### 0–1s：停住拇指

这是打断层。

系统会寻找这些信号：

- 视觉断裂
- 声音反差
- 直接人群点名
- 不可能的物体行为
- 即时痛点场景
- 社交证明突然入场
- 高具体度字幕

用户不需要立刻理解全部内容。用户只需要先别划走。

### 1–3s：证明相关

这是识别层。

Hook 必须回答：

> “为什么这个用户应该继续看？”

相关性可以来自：

- “这就是我的问题。”
- “这就是我的日常。”
- “这就是我的身份。”
- “这就是我想要的结果。”
- “这个场景太熟悉了。”
- “这个证明看起来可以验证。”

### 3–7s：产品承接

这是商业层。

产品不应该像广告一样被硬塞进来，而应该以这些角色进入：

- 缺失的工具
- 更省力的一步
- 可见证明物
- 场景解决方式
- 情绪释放点
- 下一步行动

如果产品像突然插播广告一样出现，Hook 就断了。  
如果产品像场景自然需要的东西一样出现，Hook 才成立。

---

## 5. 七类 Hook 家族

Thumb Brake 3s 使用均衡的 H1–H7 Hook 分类。这些不是死板的内容分类，而是不同的**注意力任务**。

| 类型 | 名称 | 注意力任务 | 常见失败 |
|---|---|---|---|
| H1 | 感官拦截 | 通过视觉、声音、质感、运动或节奏停住信息流 | 好看但和产品无关 |
| H2 | 张力 / 冲突 | 让用户感觉问题正在发生 | 有戏剧性但不能承接产品 |
| H3 | 好奇缺口 | 制造缺失原因、隐藏答案或延迟揭示 | 标题党，没有兑现 |
| H4 | 自我相关 | 让用户识别自己的身份、角色或日常 | “给女生”“给宝妈”这种泛标签 |
| H5 | 证明 / 结果 | 展示产品正在工作、改变、对比或解决问题 | 不可信或过度承诺 |
| H6 | 社交信号 | 使用评论、反应、同伴验证或创作者语境 | 假社证、过度表演的 UGC |
| H7 | 文化识别 | 借用熟悉的格式、仪式、梗结构或文化语法 | 抄 IP、过时趋势、复刻创作者台词 |

生成器不是简单选一个类型然后写一句话。它会抽取差异化的 Hook narrative，注入资源约束，再让模型生成三个不同的开场概念。

---

## 6. Hook 语法

大多数有效 Hook 都可以用这套语法描述：

```text
[人群或瞬间]
+ [可观察的场景证据]
+ [张力或开放环]
+ [产品承接]
+ [边界约束]
```

示例：

```text
给孩子不爱刷牙的家长
+ 睡前躲牙刷、闭嘴、家长犹豫
+ “为什么每天刷牙都像谈判？”
+ 葡萄味儿童牙膏作为更低阻力的第一步进入
+ 不承诺治疗，不羞辱，不保证效果
```

这套语法能避免 Hook 变成两种问题：

- 空泛广告文案
- 有画面但没有商业逻辑的短片段

---

## 7. 什么是好 Hook？

好 Hook 不一定很吵。它首先是**具体**。

### 弱

```text
Kids love this toothpaste!
```

### 更强

```text
Your kid does not hate brushing. They hate the first 10 seconds.
```

为什么有效：

- 它重新定义了问题。
- 它点名了具体行为。
- 它制造了一个小的好奇缺口。
- 它给产品承接留出了空间。

---

### 弱

```text
For busy moms.
```

### 更强

```text
For the parent who has already asked “go brush your teeth” four times tonight.
```

为什么有效：

- 它点名真实瞬间。
- 它避免平面化的人口标签。
- 它通过场景证据制造自我识别。

---

### 弱

```text
This cleaning spray is amazing.
```

### 更强

```text
If the stain disappears before the towel moves, people stop scrolling.
```

为什么有效：

- 它从可见证明开始。
- 它把产品动作变成视觉测试。
- 它把信息流看成一个证明环境。

---

## 8. Pattern Card 是算子，不是文案

资源库不是成品广告句子的集合。

好的 pattern card 应该像一个算子：

```text
拿到这个产品。
放进这个注意力机制。
加入这种场景证据。
避开这些失败模式。
用这种方式承接到产品。
```

差的资源设计：

```text
“这里有 50 条爆款开头。”
```

好的资源设计：

```text
“这里有 50 种制造停滑、证明相关、承接产品的方法。”
```

这一区别很重要。Thumb Brake 3s 不是复制爆款句子，而是生成可复用的创意逻辑。

---

## 9. Few-shot 是指南针，不是模板

Few-shot 用来指导节奏、具体度、结构和场景逻辑。

它不应该被逐字复制。

好的 few-shot 会教模型：

- 开头移动得有多快
- 需要什么样的视觉证据
- 如何锁住产品身份
- 产品应该什么时候进入
- 哪种证明或情绪是可接受的
- 哪些表达不应该过度承诺

当 few-shot 帮助模型生成一个属于当前产品的新 Hook，而不是近似复刻旧案例时，它才是成功的。

---

## 10. 产品承接

没有产品承接的 Hook 只是注意力。

产品可以扮演多种角色：

| 承接角色 | 含义 |
|---|---|
| 工具 | 产品帮助用户完成任务 |
| 捷径 | 产品减少步骤或降低努力 |
| 证明物 | 产品制造可见结果 |
| 舒适物 | 产品降低阻力、担忧、疼痛或犹豫 |
| 仪式物 | 产品嵌入重复日常 |
| 状态物 | 产品让用户更有准备、更有掌控感 |
| 场景解决 | 产品解决开头引出的张力 |

产品承接必须是被“挣来”的。  
如果前几秒讲的是一个问题，产品解决的是另一个问题，Hook 就断裂了。

---

## 11. 文化借用

文化可以制造即时识别。

但这个项目把文化借用理解为**结构灵感**，不是复制。

安全的文化借用包括：

- 熟悉的社交仪式
- 可识别的视频格式
- 宽泛的审美语法
- 某类类型片节奏
- 非特定的场景形状
- 抽象的梗机制

不安全的文化借用包括：

- 具体创作者原句
- 私有广告逐字稿
- 用户名
- 受版权保护角色
- 明确 IP
- 竞品创意直接复刻
- 下载的广告素材

目标不是偷一个趋势，而是理解趋势为什么能产生识别，然后把这个机制围绕自己的产品重建。

## 12. Thumb Brake 3s 如何生成 Hook

生成流程的目标是让结果既有表达力，又有结构。

```mermaid
flowchart LR
  A[商品信息] --> B[产品契约]
  B --> C[抽取 3 个 Hook narrative]
  C --> D[注入资源包]
  D --> E[生成创意规格]
  E --> F[LLM 脚本生成]
  F --> G[确定性校验]
  G --> H[定向修复]
  H --> I[未来视频 Prompt]
  I --> J[3 张 Hook 卡]
```

每个生成变体都应该回答：

```text
什么让用户停住？
为什么用户会觉得相关？
什么张力或开放环让用户继续看？
产品如何进入？
视频应该拍什么、说什么？
脚本必须避免什么？
```

---

## 13. 输出契约

一个强 Hook 输出应该包含：

```ts
type HookCard = {
  hookType: "H1" | "H2" | "H3" | "H4" | "H5" | "H6" | "H7"
  stopSignal: string
  relevanceProof: string
  sceneEvidence: string[]
  openLoop?: string
  productBridge: string
  claimBoundary: string[]
  timing: {
    "0-1s": string
    "1-3s": string
    "3-7s": string
  }
}
```

这不一定是公开 API 的实际返回形状，而是输出背后的创意契约。

## 14. 反模式

需要避免这些失败模式。

### 空泛人群标签

```text
给女生。
给年轻人。
给宝妈。
给上班族。
```

修复：

```text
不要只点名人群，要点名具体瞬间。
```

### 有标签，无场景

```text
给忙碌的 founder。
```

修复：

```text
展示这个 founder 晚上 11:48 还在 6 个标签页之间切换。
```

### 有证明，但不可见

```text
马上见效。
```

修复：

```text
展示可见前后对比、倒计时、分屏、残留、屏幕状态或真实反应。
```

### 产品瞬移

```text
有趣场景 → 产品突然出现。
```

修复：

```text
让产品解决开头引出的同一个张力。
```

### 趋势复刻

```text
复制另一个创作者的具体台词或设定。
```

修复：

```text
借结构，不借成品。
```

### 不支持的承诺

```text
保证治愈、保证投放给某人群、保证结果。
```

修复：

```text
使用可观察、符合产品和类目边界的表达。
```

## 15. 如何新增 Hook 资源

新增 pattern card、example 或 few-shot 时，先问十个问题：

1. 停滑信号是什么？
2. 用户会识别到什么？
3. 张力或开放环是什么？
4. 哪些可见场景证据支撑它？
5. 产品扮演什么角色？
6. 它适合哪些产品类目？
7. 哪些承诺或角度需要避免？
8. 它和现有 H1–H7 pattern 有什么差异？
9. 它能生成很多广告，还是只能生成一个广告？
10. 不复制趋势，它是否仍然成立？

好的资源能扩大创意范围，同时不牺牲安全、结构和产品相关性。

---

## 16. 最终原则

Hook 不是视频的开头。

Hook 是用户决定这条视频是否值得拥有“未来”的那个瞬间。

Thumb Brake 3s 存在的意义，就是设计这个瞬间。
